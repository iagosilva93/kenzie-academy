PRINCIPAIS TERMOS:

COMMITS = GRAVA AS ULTIMAS MUDANCAS NO SEU CODIGO
SALVA TUDO QUE FOI FEITO
PODEMOS SUBIR NOSSO CODIGO EM PARTE PARA O GITHUB ENVIANDO OS COMMITS E ACESSALOS DEPOIS PARA UMA CORRECAO E CONSULTAR O QUE FOI ALTERADO ENTRE UM COMMIT E OUTRO

BRANCH = SAO OS "GALHOS DA ARVORE" DO CODIGO, SAO TODOS OS GALHOS
PODEMOS TER NOSSA PROPRIAS BRANCH E IR ACOPLANDO ELA (INTEGRANDO PARTE DO CODIGO EM UM CODIGO MUITO MAIOR)
ELA QUE MANTEM O TRABALHO EM GRUPO BEM ORGANIZADO E SEM CONFLITOS

CLONE = CLONAR UM REPOSITORIO
PODEMOS COPIAR UM CODIGO DO GITHUB PARA NOSSA MAQUINA PARA ALTERAR O CODIGO/IMPLEMENTAR/ETC

PUSH = USADO PARA O SUBIR O CODIGO PARA GITHUB

PULL = PARECIDO COM O CLONE POREM SOMENTE USAMOS QUANDO O CODIGO ESTA EM NOSSA MAQUINA
APOS CLONAR UM REPOSITORIO CASO ALGUEM TENHA ATUALIZADO ALGO NO CODIGO PODEMOS DAR UM PULL NESSE CODIGO E TRAZER ESSAS ATUALIZACOES PARA NOSSA MAQUINA LOCAL
